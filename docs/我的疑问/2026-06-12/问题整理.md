# 二、HealthAgent 功能检查

## 当前问题

> HealthAgent 是否通过饮食历史及健康相关知识来分析？

需要确认：

### 1. HealthAgent输入是否完整

现在应该至少包含：

## 用户个人信息

例如：

```json
{
"user_id":"001",
"age":25,
"gender":"male",
"height":175,
"weight":70,
"activity_level":"medium",
"health_goal":"lose_weight"
}
```

来源：

```
UserProfile数据库
        |
        ↓
HealthAgent
```

需要检查：

-  是否已经保存用户信息
    
-  HealthAgent调用时是否读取
    
-  是否只是前端传入临时参数
    

---

## 2. 饮食历史是否存在

你希望：

> 每天吃的饭的信息（暂时只包括确认制作了的菜谱）

那么应该存在：

```
MealHistory表
```

例如：

|字段|说明|
|---|---|
|id|记录ID|
|user_id|用户|
|date|日期|
|recipe_id|菜谱|
|recipe_name|名称|
|calories|热量|
|ingredients|食材|
|confirmed|是否确认制作|

流程：

```
RecipeAgent生成菜谱

↓

用户点击确认制作

↓

MealHistory增加记录

↓

HealthAgent读取
```

检查：

-  有没有MealHistory数据库
    
-  用户确认制作是否写入
    
-  HealthAgent是否查询历史
    

---

## 3. 健康知识来源

HealthAgent不能只是：


```
用户信息
+
历史饮食
+
营养规则
+
健康知识库

↓

健康分析
```

例如：

```

蛋白质:
↑

蔬菜:
↓

脂肪:
↑


建议:
增加深色蔬菜摄入
```

需要检查：

-  是否有Nutrition Knowledge
    
-  是否调用RAG
    
-  是否只有LLM自由生成

# 三、InventoryAgent库存闭环检查

你的问题：

> InventoryAdjust是否被使用？

重点检查。

---

## 理想流程

### 菜谱推荐

RecipeAgent:

```
推荐:

宫保鸡丁

需要:

鸡胸肉 300g
花生 50g
黄瓜 100g

```

---

用户：

点击：

```
确认制作
```

然后：

InventoryAgent:

检查:

```
Inventory:

鸡胸肉 500g

↓

减少300g

↓

剩余200g
```

---

## 需要排查

### 1. 是否存在确认动作？

检查前端：

有没有：

```
确认制作按钮
```

没有的话：

InventoryAdjust不会发生。

---

### 2. 是否存在库存扣减接口？

例如：

```
POST

/api/inventory/adjust

{
user_id,
recipe_id
}
```

检查：

-  backend有没有接口
    
-  前端有没有调用
    
-  Agent有没有执行
    

---

### 3. 是否支持恢复

你的需求：

> 可以恢复状态

所以不能直接：

```
库存 -= 食材
```

应该：

增加操作记录：

InventoryTransaction

例如：

|id|操作|
|---|---|
|001|制作宫保鸡丁|
|002|减少鸡肉300g|

恢复：

```
rollback(transaction_id)
```
