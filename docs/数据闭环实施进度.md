# 数据闭环实施进度

> 最后更新：2026-06-12

本文档记录"用户菜谱推荐 → 确认制作 → 扣库存 → 饮食历史 → 健康分析"完整链路的实施状态。

---

## 一、已完成 ✅

### 1.1 RecipeRecommendResponse 增加结构化配料

- **文件**：`app/schemas/agent_tools.py`
- **改动**：新增 `RecipeIngredient(name, amount, unit)` 模型，`RecipeRecommendResponse` 增加 `ingredients: list[RecipeIngredient]` 字段
- **影响**：LLM 生成菜谱时同步产出结构化食材列表，为库存扣减提供数据基础

### 1.2 LLM Prompt 更新

- **文件**：`app/services/recipe_agent_service.py`
- **改动**：system prompt 要求 LLM 输出 `ingredients` 数组；模板兜底菜谱也带默认配料
- **示例输出**：`{"name": "鸡胸肉", "amount": 200, "unit": "g"}`

### 1.3 新增 MealHistory 模型

- **文件**：`app/models/meal.py`
- **表**：`meal_history`
- **字段**：`id`, `user_id`, `meal_date`, `recipe_title`, `estimated_calories`, `recipe_data`(JSON), `confirmed_at`
- **用途**：用户每次确认制作后写入一条记录

### 1.4 新增 Conversation 模型

- **文件**：`app/models/conversation.py`
- **表**：`conversations`
- **字段**：`id`, `user_id`, `conversation_date`, `messages`(JSON), `last_active_at`
- **用途**：持久化对话历史，解决页面切换后消息丢失

### 1.5 Alembic 迁移

- **文件**：`alembic/versions/20260612_04_add_meal_history_and_conversations.py`
- **操作**：`alembic upgrade head` 创建 `meal_history` 和 `conversations` 两张表

### 1.6 InventoryService 批量扣减

- **文件**：`app/services/inventory_service.py`
- **新增方法**：`batch_adjust_by_ingredients(ingredients) → BatchAdjustResult`
- **逻辑**：对每个配料模糊匹配库存项（优先扣最早过期），返回 `deducted` / `missing` / `errors`
- **类**：`BatchAdjustResult(deducted, missing, errors)`

### 1.7 Meal 确认 API

- **文件**：`app/api/routes/meal.py`
- **POST `/api/meals/confirm`**：接收 `{user_id, recipe}` → 批量扣库存 + 写 MealHistory → 返回扣减明细
- **GET `/api/meals/history?user_id=&days=7`**：查询用户饮食历史

### 1.8 HealthAgent 增强

- **文件**：`app/services/health_agent_service.py`
- **新增方法**：`analyze_with_history(payload, meal_history, days)` — 在 BMI 基础上叠加近期饮食趋势分析
- **洞察**：日均热量 vs 建议摄入对比、餐数统计、热量偏高/偏低提示

### 1.9 前端类型与 API 更新

- **文件**：`frontend/src/types/index.ts` — 增加 `RecipeIngredient` 类型
- **文件**：`frontend/src/services/api.ts` — 新增 `mealAPI.confirm()` 和 `mealAPI.history()`

### 1.10 前端确认制作按钮

- **文件**：`frontend/src/components/RecipeRecommender.tsx`
- **新增**：菜谱结果卡片中展示配料列表 + 绿色"✅ 确认制作"按钮
- **交互**：点击后调用 `/api/meals/confirm`，显示扣减结果和库存不足提示

## 三、数据闭环验证

### 验证步骤

1. 启动后端 + 数据库（`docker-compose up -d postgres`）
2. 运行迁移：`alembic upgrade head`
3. 添加库存：`POST /api/inventory` 添加鸡胸肉 500g、花生 100g 等
4. 推荐菜谱：前端 RecipeRecommender 获取推荐
5. 确认制作：点击"✅ 确认制作"
6. 验证：
   - `GET /api/inventory` 查看库存是否扣减
   - `GET /api/meals/history?user_id=test` 查看饮食记录
   - `GET /api/agents/health/analyze` 验证分析是否含历史洞察
