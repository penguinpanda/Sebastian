# 对话持久化

> 更新时间: 2026-06-12

## 1. 设计动机

前端对话页面在切换路由或刷新后会丢失消息。需要将对话历史持久化到 PostgreSQL，支持：

- 自动保存对话（前端 2s 去抖）
- 按日期加载历史对话
- 左侧历史日期栏，可切换查看不同日期的对话

## 2. 数据模型

### `Conversation` — `app/models/conversation.py`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `user_id` | String(64) | 用户 ID，有索引 |
| `conversation_date` | Date | 对话日期 |
| `messages` | JSON | 完整对话消息数组 |
| `last_active_at` | DateTime | 最后活跃时间 |

**唯一性约束:** 每个 `user_id + conversation_date` 组合唯一。同一天多次保存会覆盖 `messages`。

## 3. API 端点

### 3.1 保存对话 `POST /api/conversations/save`

**请求:**
```json
{
  "user_id": "u1",
  "date": "2026-06-12",
  "messages": [
    {"id": "1", "role": "user", "content": "你好"},
    {"id": "2", "role": "assistant", "content": "你好！有什么可以帮你的？"}
  ]
}
```

**处理逻辑:**
1. 按 `user_id + date` 查询已有记录
2. **存在:** 覆盖 `messages`，更新 `last_active_at`
3. **不存在:** 创建新记录

**响应 (200):**
```json
{
  "id": "uuid",
  "user_id": "u1",
  "date": "2026-06-12",
  "messages": [...],
  "last_active_at": "2026-06-12T10:30:00+00:00"
}
```

### 3.2 加载对话 `GET /api/conversations`

```
GET /api/conversations?user_id=u1&date=2026-06-12
```

**无记录时:**
```json
{"id": "", "user_id": "u1", "date": "2026-06-12", "messages": [], "last_active_at": ""}
```

### 3.3 对话日期列表 `GET /api/conversations/dates`

```
GET /api/conversations/dates?user_id=u1
```

**响应:**
```json
{"dates": ["2026-06-12", "2026-06-11", "2026-06-10"]}
```

按日期倒序排列，供前端左侧历史栏渲染。
