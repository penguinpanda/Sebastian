# 饮食确认与数据闭环

> 更新时间: 2026-06-12

## 1. 设计动机

用户通过 Agent 获取菜谱推荐后，需要一个"确认制作"动作来闭环：

```
菜谱推荐 → 用户确认 → 扣减库存 → 记录饮食 → 菜谱入库 → 健康分析可引用
```

这个链路将"推荐"转化为"行为数据"，为 HealthAgent 的饮食趋势分析提供数据基础。

## 2. API 端点

### 2.1 确认制作 `POST /api/meals/confirm`

**请求:**
```json
{
  "user_id": "u1",
  "recipe": {
    "title": "宫保鸡丁",
    "rationale": "...",
    "estimated_calories": 600,
    "ingredients": [
      {"name": "鸡胸肉", "amount": 200, "unit": "g"},
      {"name": "花生", "amount": 50, "unit": "g"}
    ],
    "steps": ["切鸡胸肉", "炒制"],
    "required_equipment": ["pan"],
    "missing_ingredients": [],
    "feasible": true,
    "missing_equipment": []
  }
}
```

**响应 (201):**
```json
{
  "meal_id": "uuid",
  "status": "confirmed",
  "deducted": [
    {"name": "鸡胸肉", "amount": 200, "unit": "g", "inventory_name": "鸡胸肉", "inventory_id": "uuid"}
  ],
  "missing": [],
  "errors": []
}
```

**处理流程:**

```mermaid
flowchart TD
    A[POST /meals/confirm] --> B[batch_adjust_by_ingredients]
    B --> C{库存足够?}
    C -->|是| D[扣减成功 → deducted[]]
    C -->|部分| E[部分扣减 → deducted[] + missing[]]
    C -->|否| F[全部缺失 → missing[]]
    D --> G[写 MealHistory]
    E --> G
    F --> G
    G --> H[MD5 去重写 Recipe]
    H --> I[返回 201]
```

### 2.2 查询饮食历史 `GET /api/meals/history`

```
GET /api/meals/history?user_id=u1&days=7
```

返回最近 N 天的饮食记录列表。

### 2.3 回退餐食 `POST /api/meals/{meal_id}/rollback`

回退已确认的餐食，将扣减的库存恢复。约束：
- 只能回退未回退过的记录（`rolled_back_at IS NULL`）
- 已回退的记录再次请求返回 409

## 3. 库存批量扣减

### `InventoryService.batch_adjust_by_ingredients()`

```python
def batch_adjust_by_ingredients(
    self, ingredients: list[RecipeIngredient], user_id: str | None = None
) -> BatchAdjustResult:
```

**匹配策略:** 模糊匹配（`target in item.name` 或 `item.name in target`），优先扣最早过期的批次。

**返回结构:**
```python
@dataclass
class BatchAdjustResult:
    deducted: list[dict]  # [{name, amount, unit, inventory_name, inventory_id}]
    missing: list[dict]   # [{name, amount, unit}]
    errors: list[str]
```

## 4. 数据模型

### `MealHistory` — `app/models/meal.py`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `user_id` | String(64) | 用户 ID，有索引 |
| `meal_date` | Date | 用餐日期 |
| `recipe_title` | String(200) | 菜谱标题 |
| `estimated_calories` | Integer | 预估热量 |
| `recipe_data` | JSON | 完整 RecipeRecommendResponse |
| `deducted_detail` | JSON | 扣减明细 `[{name, amount, unit, inventory_name, inventory_id}]` |
| `rolled_back_at` | DateTime | 回退时间（NULL 表示未回退） |
| `confirmed_at` | DateTime | 确认时间 |

**用途:** HealthAgent 查询用户近期饮食历史，分析日均热量摄入和饮食趋势。

## 5. 菜谱自动入库

确认制作时，系统自动将菜谱写入 `recipes` 表：

- **去重:** MD5(title + 食材名排序) 作为 `content_hash`
- **重复制作:** `times_made += 1`
- **新菜谱:** 创建新记录，`times_made = 1`

详见 [菜谱库](./菜谱库.md)。

## 6. 与 HealthAgent 的集成

`POST /api/agents/health/analyze` 在处理请求时：

1. 从 `meal_history` 查询用户近 7 天饮食记录
2. 将饮食历史注入 `HealthAgent.analyze()` 的 `meal_history` 参数
3. 调用 `analyze_with_history()` 方法，在 BMI 基础上叠加：
   - 日均热量 vs 建议摄入对比
   - 餐数统计
   - 热量偏高/偏低提示
