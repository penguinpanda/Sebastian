# 菜谱库

> 更新时间: 2026-06-12

## 1. 设计动机

每次推荐菜谱依赖 LLM 生成，成本高且结果不稳定。将用户确认过的菜谱自动入库，可实现：

- **快速检索:** 用户可直接搜索历史菜谱，无需 LLM 重新生成
- **去重:** 同一菜谱多次制作只存一条记录，`times_made` 递增
- **热门排行:** 按制作次数排序，展示用户最常做的菜
- **记忆检索:** SearchAgent 可从菜谱库检索相关记忆

## 2. 数据模型

### `Recipe` — `app/models/recipe.py`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `title` | String(200) | 菜谱名称 |
| `user_id` | String(64) | 用户 ID，有索引 |
| `estimated_calories` | Integer | 预估热量 |
| `ingredients` | JSON | `[{name, amount, unit}]` |
| `steps` | JSON | `["步骤1", "步骤2"]` |
| `required_equipment` | JSON | `["pan", "pot"]` |
| `recipe_data` | JSON | 完整 RecipeRecommendResponse |
| `times_made` | Integer | 制作次数，默认 1 |
| `content_hash` | String(32) | MD5 去重指纹，有索引 |

## 3. 自动入库机制

在 `POST /api/meals/confirm` 中，确认制作后自动触发：

```python
# 构造 MD5 指纹: title + 排序后的食材名
content = f"{recipe.title}|{'|'.join(sorted(i.name for i in recipe.ingredients))}"
recipe_hash = hashlib.md5(content.encode()).hexdigest()

existing = db.execute(
    select(Recipe).where(Recipe.content_hash == recipe_hash)
).scalars().first()

if existing:
    existing.times_made += 1     # 重复制作：计数 +1
else:
    db.add(Recipe(...))          # 新菜谱：创建记录
```

## 4. API 端点

### 4.1 搜索菜谱 `GET /api/recipes`

```
GET /api/recipes?user_id=u1&query=鸡&sort=times_made&limit=20
```

**参数:**

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `user_id` | string | 必填 | 用户 ID |
| `query` | string | `""` | 标题模糊搜索 |
| `sort` | enum | `times_made` | 排序: `times_made` / `calories` / `recent` |
| `limit` | int | 20 | 返回条数 (1-100) |

**响应:**
```json
{
  "recipes": [
    {
      "id": "uuid",
      "title": "宫保鸡丁",
      "estimated_calories": 600,
      "ingredients": [{"name": "鸡胸肉", "amount": 200, "unit": "g"}],
      "steps": ["切鸡胸肉", "炒制"],
      "times_made": 3,
      "created_at": "2026-06-12T10:00:00+00:00"
    }
  ],
  "count": 1
}
```

### 4.2 热门菜谱 `GET /api/recipes/top`

```
GET /api/recipes/top?user_id=u1&limit=10
```

按 `times_made DESC, created_at DESC` 排序，返回用户最常制作的 Top N 菜谱。

**用途:** 首页快捷推荐"你常做的菜"。
