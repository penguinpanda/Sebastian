# 代码注释导览

本文档说明本轮中文注释覆盖的核心代码路径，便于后续阅读和继续补充。

## 后端入口与横切能力

- `app/main.py`：应用启动、CORS、`trace_id` 中间件和访问日志。
- `app/api/router.py`：所有业务路由统一挂载到 `/api` 前缀下。
- `app/workers/celery_app.py`：Celery worker、broker/backend 和定时任务配置。

## 库存业务链路

- `app/api/routes/inventory.py`：库存 HTTP 接口和临期扫描任务入口。
- `app/services/inventory_service.py`：库存业务服务，负责输入归一化和 DTO 转换。
- `app/repositories/inventory.py`：内存仓储、PostgreSQL 仓储，以及“同批次自动合并”和出入库流水规则。

## Agent 与 MCP 链路

- `app/api/routes/agent.py`：聊天入口、限流、任务状态缓存和工具调用日志。
- `app/api/routes/agent_tools.py`：菜谱、健康、厨具、搜索等专用 Agent 的兼容 REST 入口。
- `app/orchestration/graph.py`：库存 Agent 的 LangGraph 主流程。
- `app/orchestration/agent_graphs.py`：专用 Agent 的 LangGraph 编排，MCP 工具和 REST 入口共用这里。
- `app/orchestration/nodes.py`：输入清洗、意图识别、回复生成和兜底回复节点。
- `app/schemas/agent_tools.py`：专用 Agent 请求和响应模型。
- `app/services/mcp_adapter.py`：MCP 工具统一调用、幂等缓存、鉴权和审计字段。

## 记忆检索链路

- `app/services/search_service.py`：Elasticsearch 索引创建、记忆写入、词法/向量/混合检索、RRF 融合排序。

## 前端入口

- `frontend/src/App.tsx`：全局标签页导航和 `userId` 传递。
- `frontend/src/services/api.ts`：Axios 实例、错误格式化、各类 API 封装和 MCP invoke 协议。
- `frontend/src/components/InventoryManager.tsx`：库存列表、临期提醒、创建/调整/删除库存的行级操作状态。

## 注释原则

- 解释模块职责、数据流向和关键业务规则。
- 对容易误解的兜底逻辑、缓存逻辑、异步任务逻辑补充“为什么”。
- 不逐行翻译代码，避免注释重复代码本身。
