# Sebastian 测试与质量

更新时间: 2026-06-11

## 1. 目标

本文档定义 Dev Ready 阶段的测试执行方式、验收标准与回归最小集合。

## 2. 测试分层

1. 单元与接口测试
   - 命令: `make test-unit`
   - 范围: API 路由、服务逻辑、图编排、缓存、搜索、MCP。
2. 集成测试
   - 命令: `make integration`
   - 范围: PostgreSQL 容器、readiness、MCP 依赖路径。
3. 全量测试
   - 命令: `make test`

## 3. Dev Ready 通过标准

- [ ] `make test-unit` 通过。
- [ ] `make integration` 在依赖可用时通过。
- [ ] 关键链路人工验收通过（见第 4 节）。

## 4. 关键链路回归清单

1. 库存链路
   - 创建库存
   - 查询库存
   - 调整库存
2. Agent 链路
   - 发起聊天
   - 查询任务状态
   - 查询队列状态
3. 搜索链路
   - 写入记忆
   - hybrid 检索
4. MCP 链路
   - 列出 tools
   - invoke 工具
5. 异步链路
   - 触发库存扫描任务
   - 查询任务执行历史

## 5. 常用命令

```powershell
make test
make test-unit
make integration
.\.conda\python.exe -m pytest -m "not integration" -v
```

## 6. 质量门禁建议

1. 提交前至少执行 `make test-unit`。
2. 修改数据库相关逻辑时，必须执行 `make integration`。
3. 修改文档中的命令或接口后，必须做一次冷启动验证。
4. 变更 Agent 或 MCP 协议时，补充对应测试用例。

## 7. 已知缺口

- 缺少完整前后端 E2E 自动化。
- 缺少性能压测基线（并发与延迟）。
- 缺少认证/鉴权相关测试矩阵。
