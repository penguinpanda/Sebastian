# 部署说明

> 本地开发与 Docker Compose 部署。

## 环境矩阵

| 组件 | 版本 | 用途 |
|------|------|------|
| Python | 3.11+（推荐 3.12） | 后端运行时 |
| Node.js | 18+ | 前端构建 |
| PostgreSQL | 16 | 事务数据库 |
| Redis | 7 | 缓存 / 队列 |
| Elasticsearch | 8.14 | 记忆检索 |
| Docker Desktop | 最新 | 容器编排 |

## 环境变量

从 `.env.example` 复制：

```powershell
Copy-Item .env.example .env
```

### 最小必填

```env
DEEPSEEK_API_KEY=sk-xxx
DATABASE_URL=postgresql+psycopg://sebastian:sebastian@localhost:5432/sebastian
REDIS_URL=redis://localhost:6379/0
ELASTICSEARCH_URL=http://localhost:9200
```

### 完整变量说明

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `APP_NAME` | Sebastian | 应用名称 |
| `APP_ENV` | development | 环境标识 |
| `LOG_LEVEL` | INFO | 日志级别 |
| `CORS_ORIGINS` | localhost:5173 | 前端 CORS |
| `DEEPSEEK_API_KEY` | （空） | LLM API Key |
| `DEEPSEEK_MODEL` | deepseek-chat | LLM 模型 |
| `LLM_TIMEOUT_MS` | 5000 | LLM 超时 |
| `LLM_RETRY_MAX` | 2 | LLM 重试 |
| `DATABASE_URL` | localhost PG | 数据库连接 |
| `REDIS_URL` | localhost:6379/0 | Redis |
| `ELASTICSEARCH_URL` | localhost:9200 | ES 地址 |
| `CELERY_BROKER_URL` | localhost:6379/1 | Celery broker |
| `CELERY_RESULT_BACKEND` | localhost:6379/2 | Celery 结果 |
| `AGENT_RATE_LIMIT_MAX_REQUESTS` | 20 | 限流上限 |
| `AGENT_RATE_LIMIT_WINDOW_SECONDS` | 60 | 限流窗口 |

## 本地开发部署

### 1. Python 环境

```powershell
conda create -p ./.conda python=3.12 -y
conda activate ./.conda
python -m pip install -e .[dev]
```

### 2. 基础设施

```powershell
docker compose up -d postgres redis elasticsearch
```

### 3. 数据库迁移

```powershell
alembic upgrade head
```

### 4. 启动 API

```powershell
.\.conda\python.exe -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 5. 启动 Celery（可选）

```powershell
.\.conda\python.exe -m celery -A app.workers.celery_app:celery_app worker --loglevel=INFO
.\.conda\python.exe -m celery -A app.workers.celery_app:celery_app beat --loglevel=INFO
```

### 6. 启动前端（可选）

```powershell
cd frontend
npm install
npm run dev
```

## Docker Compose 一键部署

```powershell
docker compose up -d
```

服务启动顺序：

```
postgres (healthcheck)
    → migrate (alembic upgrade head)
        → api (uvicorn)
        → celery-worker
        → celery-beat
redis + elasticsearch (并行)
```

### 查看状态

```powershell
docker compose ps
docker compose logs -f api
```

### 停止

```powershell
docker compose down
```

## Dev Ready 验收清单

### 服务可用性

- [ ] `GET /api/health` → `{"status":"ok"}`
- [ ] `GET /api/health/readiness` → 200
- [ ] `GET /api/health/dependencies` → 各依赖 connected

### 核心链路

- [ ] `POST /api/inventory` 写入成功
- [ ] `POST /api/agent/chat` 返回 reply + task_id
- [ ] `POST /api/agents/recipe/recommend` 返回菜谱
- [ ] `POST /api/search/memory` + `GET /api/search/memory` 可用
- [ ] `POST /api/mcp/invoke` 可调用

### 测试

- [ ] `make test-unit` 通过
- [ ] `make integration` 通过（依赖可用时）

### 数据

- [ ] `alembic upgrade head` 幂等
- [ ] PostgreSQL 可查询库存和任务

## 常见部署问题

| 问题 | 排查 |
|------|------|
| readiness 503 | `docker compose ps` 检查 PG/Redis/ES |
| Alembic 失败 | 确认 `DATABASE_URL` 和 PG 容器健康 |
| Agent 超时 | 检查 `DEEPSEEK_API_KEY` 和网络 |
| ES 启动慢 | 等待 30-60s，检查 `docker compose logs elasticsearch` |
| 端口冲突 | 修改 `docker-compose.yml` 端口映射 |

## 非目标（Dev Ready 范围外）

- 生产级高可用
- HTTPS / 认证鉴权
- Prometheus 指标导出
- 多节点 Celery 集群
