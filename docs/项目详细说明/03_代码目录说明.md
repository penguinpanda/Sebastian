# 代码目录说明

> 每个目录为什么存在、允许做什么、禁止做什么。

## 顶层结构

```
Sebastian/
├── app/              # 后端核心代码
├── frontend/         # React 前端
├── tests/            # 后端测试
├── alembic/          # 数据库迁移
├── examples/         # 新人示例脚本
├── docs/             # 文档
├── docker-compose.yml
├── pyproject.toml
└── .env.example
```

## app/ 目录详解

```
app/
├── agents/           # Agent 入口层
├── api/              # HTTP 接口层
├── core/             # 全局配置与基础设施
├── db/               # 数据库 Session 与 Base
├── integrations/     # 外部客户端（Redis、ES）
├── llm/              # LLM 客户端与 Prompt
├── models/           # SQLAlchemy ORM
├── orchestration/    # LangGraph 编排
├── repositories/     # 数据访问层
├── schemas/          # Pydantic 模型
├── services/         # 业务逻辑层
├── tasks/            # Celery 任务定义
├── workers/          # Celery 应用配置
└── main.py           # FastAPI 入口
```

---

## agents/

**职责：** 定义系统智能能力入口，是对外暴露的 Agent 门面。

**现有 Agent：**

| 文件 | 类 | 方法 |
|------|-----|------|
| `recipe_agent.py` | `RecipeAgent` | `recommend()` |
| `health_agent.py` | `HealthAgent` | `analyze()` |
| `equipment_agent.py` | `EquipmentAgent` | `check()` |
| `search_agent.py` | `SearchAgent` | `answer()` |
| `inventory_agent.py` | `InventoryAgent` | （占位，实际走 graph.py） |

**允许：**

- 调用 `orchestration` 中的 `run_*_agent()` 函数
- 持有 Service 实例（通过构造函数注入）
- 做请求/响应的类型转换

**禁止：**

- 写 SQL 或直接操作数据库
- 编写复杂业务逻辑（应放 Service）
- 直接调用 LLM（应放 Service 或 orchestration nodes）

**修改功能时：**

- 改 Agent 对外方法签名 → 这里
- 改 Agent 内部业务 → `services/*_agent_service.py`
- 改 Agent 流程顺序 → `orchestration/agent_graphs.py`

---

## api/

**职责：** 对外 HTTP 接口，请求校验与 HTTP 语义转换。

```
api/
├── router.py         # 汇总所有路由，统一 /api 前缀
├── dependencies.py   # FastAPI 依赖注入
└── routes/
    ├── agent.py          # /api/agent/chat 聊天入口
    ├── agent_tools.py    # /api/agents/* 专用 Agent
    ├── inventory.py      # /api/inventory/* CRUD
    ├── search.py         # /api/search/memory
    ├── mcp.py            # /api/mcp/*
    ├── health.py         # 健康检查
    └── task_history.py   # Celery 任务历史
```

**允许：**

- 参数校验（Pydantic Schema）
- 调用 Agent / Service
- 将业务异常转为 HTTPException
- 设置 `request.state.user_id` / `action` 供日志使用

**禁止：**

- 编写业务规则
- 直接 SQL 查询
- 在路由里写 LangGraph 节点逻辑

---

## orchestration/

**职责：** LangGraph 工作流编排，控制 Agent 执行顺序。

| 文件 | 用途 |
|------|------|
| `graph.py` | Inventory 聊天主图（normalize → intent → response） |
| `agent_graphs.py` | 专用 Agent 子图（recipe / health / equipment / search） |
| `nodes.py` | Inventory 图的节点实现 |
| `state.py` | Inventory 图的状态 TypedDict |

**允许：**

- 定义 Graph 节点与边
- 在节点内调用 Service
- 子图嵌套（Recipe 图内嵌 Search 子图）

**禁止：**

- 直接访问 Repository / 数据库
- 在节点里写 HTTP 相关逻辑

---

## services/

**职责：** 核心业务逻辑，是大部分"改功能"的目标位置。

| 文件 | 职责 |
|------|------|
| `recipe_agent_service.py` | 菜谱生成（LLM + 模板降级） |
| `health_agent_service.py` | BMI 计算与建议 |
| `equipment_agent_service.py` | 厨具集合差集校验 |
| `search_agent_service.py` | 记忆检索 + 摘要 |
| `inventory_service.py` | 库存 CRUD 与汇总 |
| `search_service.py` | Elasticsearch 检索实现 |
| `mcp_adapter.py` | MCP 协议适配 |
| `agent_rate_limiter.py` | Agent 限流 |
| `agent_task_cache.py` | Redis 任务状态缓存 |

**允许：**

- 调用 Repository、LLM、外部集成
- 实现业务规则与降级策略
- 抛出领域异常（`NotFoundError` 等）

**禁止：**

- 感知 HTTP 请求/响应
- 定义 LangGraph 节点

---

## repositories/

**职责：** 数据访问抽象，隔离 SQL 细节。

当前实现：

- `inventory.py` → `PostgresInventoryRepository`

**允许：**

- SQLAlchemy 查询与写入
- 返回 ORM 模型或领域对象

**禁止：**

- 业务规则（如"BMI 分类"）
- 调用 LLM

---

## models/

**职责：** SQLAlchemy ORM 表定义。

| 表 | 模型 |
|----|------|
| `inventories` | `Inventory` |
| `inventory_transactions` | `InventoryTransaction` |
| `agent_tasks` | `AgentTask` |
| `tool_call_logs` | `ToolCallLog` |
| `celery_task_execution_logs` | `CeleryTaskExecutionLog` |

Schema 变更需同步 `alembic/versions/` 迁移文件。

---

## schemas/

**职责：** Pydantic 请求/响应模型，API 与 Service 之间的契约。

| 文件               | 内容                                                      |
| ---------------- | ------------------------------------------------------- |
| `agent_tools.py` | Recipe / Health / Equipment / Search 的 Request/Response |
| `inventory.py`   | 库存 CRUD Schema                                          |
| `search.py`      | 记忆检索 Schema                                             |
| `mcp.py`         | MCP 协议 Schema                                           |

新增 API 时，**先定义 Schema，再写路由**。

---

## llm/

**职责：** 大模型调用基础设施。

| 文件 | 用途 |
|------|------|
| `client.py` | DeepSeek HTTP 客户端（超时、重试） |
| `prompts.py` | Inventory 意图识别 Prompt 模板 |
| `output_parser.py` | LLM JSON 输出解析 |

---

## core/

**职责：** 全局配置与横切关注点。

| 文件 | 用途 |
|------|------|
| `config.py` | 环境变量 Settings |
| `errors.py` | 统一异常类 |
| `logging_utils.py` | 结构化日志 |
| `request_context.py` | trace_id 上下文 |

---

## integrations/

**职责：** 第三方客户端封装。

- `redis_client.py` — Redis 连接
- `elasticsearch_client.py` — ES 连接

---

## tasks/ + workers/

**职责：** Celery 异步任务。

- `workers/celery_app.py` — Celery 应用配置
- `tasks/inventory_tasks.py` — 临期扫描等任务

---

## frontend/

React + TypeScript 前端，通过 `frontend/src/services/api.ts` 调用后端 `/api/*`。

主要页面组件在 `frontend/src/components/`。
