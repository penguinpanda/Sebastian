# Sebastian 项目目录结构说明

## 1. 项目整体架构

项目采用分层架构设计，将：

* API 接口
* Agent 智能体
* 工作流编排
* 业务服务
* 数据访问
* 数据模型
* 大模型调用
* 后台任务

进行解耦。

整体调用流程：

```
用户请求
   |
   v
API层
(app/api)
   |
   v
Agent入口
(app/agents)
   |
   v
Agent编排
(app/orchestration)
   |
   v
业务服务
(app/services)
   |
   +----------------+
   |                |
   v                v
数据库             LLM
(db/repositories)  (llm)
```

# 1. agents

## 作用

> 定义系统中的智能 Agent，是业务能力的入口层。

例如：

```
agents
│
├── recipe_agent.py
├── equipment_agent.py
├── health_agent.py
├── inventory_agent.py
└── search_agent.py
```

对应：

| Agent          | 功能   |
| -------------- | ---- |
| RecipeAgent    | 菜谱生成 |
| EquipmentAgent | 厨具检查 |
| HealthAgent    | 健康分析 |
| InventoryAgent | 库存管理 |
| SearchAgent    | 信息搜索 |

示例：

```python
class EquipmentAgent:

    def check(self, payload):
        return run_equipment_agent(payload)
```

Agent 本身通常不处理复杂逻辑，而负责：

* 接收请求
* 调用流程
* 返回结果

调用：

```
API
 |
 v
EquipmentAgent
 |
 v
EquipmentAgentService
```

# 8. orchestration

## 作用

> Agent流程编排层。

这是你的项目比较核心的一层。

例如：

```
orchestration
|
└── agent_graphs.py
```

负责：

多个 Agent 如何协作。

例如：

用户：

> 推荐晚餐

流程：

```
用户
 |
 v
RecipeAgent
 |
 +------------+
 |            |
 v            v
HealthAgent   InventoryAgent
 |
 v
EquipmentAgent
 |
 v
生成菜谱
```

类似 LangGraph 工作流。

例如：

```python
run_equipment_agent()
```

控制：

```
节点1
 |
节点2
 |
节点3
```

# 这个项目的核心思想

一句话总结：

> **Agent负责“决定做什么”，Orchestration负责“怎么协作”，Service负责“具体怎么做”，Repository负责“怎么取数据”，LLM负责“智能生成”。**

这种结构适合扩展多智能体系统，例如：

```
Recipe Agent
       |
       +---- Health Agent
       |
       +---- Inventory Agent
       |
       +---- Equipment Agent
       |
       +---- Search Agent
```

你的项目目前已经具备一个比较完整的 Agent 应用后端架构。
