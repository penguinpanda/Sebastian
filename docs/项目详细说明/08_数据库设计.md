# 数据库设计

> 当前已落地的表结构与数据边界。

## 设计原则

1. **PostgreSQL 是事务真相源** — 库存、任务、日志
2. **Elasticsearch 是检索层** — 记忆索引，不承担事务一致性
3. **Redis 是缓存层** — 限流、任务状态、Celery 队列

Schema 权威来源：`app/models/` + `alembic/versions/`

## 表说明

### inventories

库存主表。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | VARCHAR(120) | 食材名称 |
| quantity | NUMERIC(12,3) | 当前数量 |
| unit | VARCHAR(20) | 单位 |
| expire_date | DATE | 过期日期 |
| note | TEXT | 备注 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

模型：`app/models/inventory.py` → `Inventory`

### inventory_transactions

库存变动流水。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| inventory_id | UUID | FK → inventories.id (CASCADE) |
| action | VARCHAR(20) | 操作类型 |
| amount | NUMERIC(12,3) | 变动量（正=补货，负=消耗） |
| note | TEXT | 备注 |
| created_at | TIMESTAMPTZ | 创建时间 |

模型：`app/models/inventory.py` → `InventoryTransaction`

### agent_tasks

Agent 任务记录（聊天等）。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | VARCHAR(64) | 用户标识（可空） |
| task_type | VARCHAR(40) | 任务类型，如 `inventory_chat` |
| status | VARCHAR(20) | pending / running / completed / failed |
| input_payload | JSON | 输入 |
| output_payload | JSON | 输出或错误 |
| created_at | TIMESTAMPTZ | 创建时间 |

模型：`app/models/agent.py` → `AgentTask`

### tool_call_logs

Agent 工具调用日志。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| task_id | UUID | FK → agent_tasks.id |
| trace_id | VARCHAR(64) | 请求追踪 ID |
| tool_name | VARCHAR(80) | 工具名称 |
| latency_ms | INTEGER | 耗时 |
| result_status | VARCHAR(20) | ok / error |
| error_detail | TEXT | 错误详情 |
| created_at | TIMESTAMPTZ | 创建时间 |

模型：`app/models/agent.py` → `ToolCallLog`

### celery_task_execution_logs

Celery 异步任务执行日志。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| celery_task_id | VARCHAR(64) | Celery 任务 ID |
| task_name | VARCHAR(120) | 任务名称 |
| status | VARCHAR(20) | 执行状态 |
| trace_id | VARCHAR(64) | 追踪 ID |
| input_payload | JSON | 输入 |
| output_payload | JSON | 输出 |
| error_detail | TEXT | 错误 |
| started_at | TIMESTAMPTZ | 开始时间 |
| finished_at | TIMESTAMPTZ | 结束时间 |

模型：`app/models/task_execution.py` → `CeleryTaskExecutionLog`

## Elasticsearch 索引

### memory_index

记忆检索索引，不通过 SQLAlchemy 管理。

| 概念 | 说明 |
|------|------|
| 写入 | `POST /api/search/memory` |
| 检索 | `GET /api/search/memory` |
| 模式 | lexical / vector / hybrid (RRF) |
| 配置 | `ELASTICSEARCH_MEMORY_INDEX` 环境变量 |

## 迁移管理

```powershell
# 查看当前版本
alembic current

# 执行迁移
alembic upgrade head

# 创建新迁移（修改 models/ 后）
alembic revision --autogenerate -m "describe_change"
```

迁移文件位于 `alembic/versions/`：

| 文件 | 内容 |
|------|------|
| `20260609_01_init_inventory_tables.py` | inventories + inventory_transactions |
| `20260609_02_add_agent_task_and_tool_call_log.py` | agent_tasks + tool_call_logs |
| `20260611_03_add_celery_task_execution_logs.py` | celery_task_execution_logs |

## 新增表步骤

1. 在 `app/models/` 定义 ORM 模型
2. 在 `app/models/__init__.py` 导出
3. 运行 `alembic revision --autogenerate`
4. 检查生成的迁移 SQL
5. 执行 `alembic upgrade head`
6. 如需 Repository，在 `app/repositories/` 添加
7. 更新本文档

## Redis 数据结构

Redis 不存储持久业务数据，仅用于：

| 用途 | Key 模式 | TTL |
|------|----------|-----|
| Agent 限流 | 滑动窗口计数 | 窗口期 |
| 任务状态缓存 | task status | `AGENT_TASK_STATUS_TTL_SECONDS` |
| MCP 幂等 | idempotency key | `MCP_IDEMPOTENCY_TTL_SECONDS` |
| Celery broker | 队列 | 无 |

## 数据边界速查

| 数据 | 存哪里 | 不要存哪里 |
|------|--------|-----------|
| 库存数量 | PostgreSQL | Elasticsearch |
| 用户记忆 | Elasticsearch | PostgreSQL |
| 任务状态（短期） | Redis + PostgreSQL | 仅 Redis |
| Agent 调用审计 | PostgreSQL (tool_call_logs) | Redis |
