# 快速开始

> 新人拿到代码后，15 分钟内完成本地启动。

## 环境要求

| 依赖 | 版本 |
|------|------|
| Python | >= 3.11（推荐 3.12） |
| Node.js | 18+（仅前端） |
| Docker Desktop | 最新稳定版 |
| conda | 推荐（也可用 venv + pip） |

基础设施（通过 Docker Compose 启动）：

- PostgreSQL 16
- Redis 7
- Elasticsearch 8.x（搜索/记忆功能需要）

## 安装

```powershell
git clone <repo-url>
cd Sebastian

# 创建 Python 环境
conda create -p ./.conda python=3.12 -y
conda activate ./.conda
python -m pip install -e .[dev]
```

## 配置

复制环境变量模板并填写必要项：

```powershell
Copy-Item .env.example .env
```

最小必填项：

```env
DEEPSEEK_API_KEY=sk-xxx          # Agent 调用 LLM 所需
DATABASE_URL=postgresql+psycopg://sebastian:sebastian@localhost:5432/sebastian
REDIS_URL=redis://localhost:6379/0
ELASTICSEARCH_URL=http://localhost:9200
```

完整变量说明见 `.env.example` 与 [11_部署说明.md](./11_部署说明.md)。

## 启动基础设施

```powershell
docker compose up -d postgres redis elasticsearch
alembic upgrade head
```

## 启动后端

```powershell
.\.conda\python.exe -m uvicorn app.main:app --reload
```

默认地址：`http://127.0.0.1:8000`

验证：

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/health
Invoke-RestMethod http://127.0.0.1:8000/api/health/readiness
```

## 启动前端（可选）

```powershell
cd frontend
npm install
npm run dev
```

默认地址：`http://127.0.0.1:5173`

前端 API 地址在 `frontend/.env.development`：

```env
VITE_API_BASE_URL=http://127.0.0.1:8000/api
```

## 启动 Celery（可选）

临期扫描等异步任务需要 worker：

```powershell
.\.conda\python.exe -m celery -A app.workers.celery_app:celery_app worker --loglevel=INFO
.\.conda\python.exe -m celery -A app.workers.celery_app:celery_app beat --loglevel=INFO
```

## 一键 Docker Compose

```powershell
docker compose up -d
```

会自动执行 migrate → api → celery-worker → celery-beat。

## 测试

```powershell
# 全部单元测试（不依赖 Docker）
make test-unit

# 含集成测试（需要 PostgreSQL/Redis/ES 可用）
make integration

# 或直接
.\.conda\python.exe -m pytest -m "not integration" -v
```

## 快速体验 Agent

```powershell
# 菜谱推荐
Invoke-RestMethod -Method POST `
  -Uri http://127.0.0.1:8000/api/agents/recipe/recommend `
  -ContentType "application/json" `
  -Body '{"user_id":"demo","meal_type":"dinner","target_calories":600}'

# 库存聊天
Invoke-RestMethod -Method POST `
  -Uri http://127.0.0.1:8000/api/agent/chat `
  -ContentType "application/json" `
  -Body '{"message":"我有哪些临期食材？","user_id":"demo"}'
```

也可运行 `examples/` 目录下的示例脚本，见 [examples/README.md](../examples/README.md)。

## 常见问题

| 现象 | 排查 |
|------|------|
| readiness 503 | 检查 postgres / redis / elasticsearch 容器是否运行 |
| Agent 超时 | 检查 `DEEPSEEK_API_KEY`，适当调高 `LLM_TIMEOUT_MS` |
| 前端连不上后端 | 检查 `VITE_API_BASE_URL` 与 `CORS_ORIGINS` |

更多问题见 [12_FAQ.md](./12_FAQ.md)。
