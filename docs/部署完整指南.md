# Sebastian 部署完整指南

更新时间: 2026-06-11

## 1. 目标与范围

本文档定义 Sebastian 在本地/测试环境达到 Dev Ready 的部署标准，不覆盖生产级高可用与安全合规。

## 2. 依赖矩阵

- Python: 3.12+
- Node.js: 18+
- Docker Desktop: 最新稳定版
- PostgreSQL: 16（docker-compose 默认）
- Redis: 7（docker-compose 默认）
- Elasticsearch: 8.x（可选，搜索相关能力依赖）

## 3. 环境变量最小集合

建议从 `.env.example` 复制 `.env` 后再修改。

- `DEEPSEEK_API_KEY`: 必填，Agent 调用 LLM 所需。
- `DATABASE_URL`: 默认本地 PostgreSQL。
- `REDIS_URL`: 默认本地 Redis db0。
- `ELASTICSEARCH_URL`: 默认 `http://localhost:9200`。
- `DEEPSEEK_MODEL`: 默认 `deepseek-chat`。
- `MCP_AUTH_ENABLED`: 默认 false（Dev 场景）。
- `AGENT_RATE_LIMIT_MAX_REQUESTS`: 默认 20。
- `AGENT_RATE_LIMIT_WINDOW_SECONDS`: 默认 60。

## 4. 冷启动流程

1. 创建并激活 Python 环境。
2. 安装项目依赖。
3. 启动基础依赖（PostgreSQL/Redis）。
4. 执行 Alembic 迁移。
5. 启动 API。
6. 按需启动 Celery worker 与 beat。

命令统一参考 `docs/使用指南.md`。

## 5. Docker Compose 一键启动

执行:

```bash
docker compose up -d
```

默认服务顺序:

1. postgres（健康检查通过）
2. migrate（执行 Alembic）
3. api
4. celery-worker
5. celery-beat
6. redis

## 6. Dev Ready 验收清单

### 6.1 服务可用性

- [ ] `GET /api/health` 返回 `{"status":"ok"}`。
- [ ] `GET /api/health/dependencies` 能返回依赖状态。
- [ ] `GET /api/health/readiness` 依赖可用时返回 200。

### 6.2 核心链路

- [ ] `POST /api/inventory` 成功写入数据。
- [ ] `POST /api/agent/chat` 成功创建任务并返回回答。
- [ ] `GET /api/agent/tasks/{task_id}` 返回运行状态。
- [ ] `POST /api/search/memory` 与 `GET /api/search/memory` 可用。
- [ ] `POST /api/mcp/invoke` 可调用已注册工具。

### 6.3 数据与迁移

- [ ] `alembic upgrade head` 可重复执行（幂等）。
- [ ] PostgreSQL 可查询到新增库存与任务日志。

### 6.4 异步任务

- [ ] 触发 `POST /api/inventory/tasks/scan-expiring` 能成功入队。
- [ ] worker 日志可见任务执行记录。

## 7. 常见部署问题

1. readiness 返回 503
   - 检查 PostgreSQL/Redis/Elasticsearch 是否可连接。
   - 使用 `docker compose ps` 查看容器状态。
2. Alembic 迁移失败
   - 检查 `DATABASE_URL` 是否指向正确数据库。
   - 确认 postgres 容器已通过 healthcheck。
3. Agent Chat 超时
   - 检查 `DEEPSEEK_API_KEY` 与外网访问。
   - 适当调整 `LLM_TIMEOUT_MS`。

## 8. 非目标说明

以下项不在 Dev Ready 范围内:

- 生产环境 SLA/SLO
- 多副本部署与自动扩缩容
- 完整 RBAC 与审计合规
- 全量告警平台接入
