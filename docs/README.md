## 3. Dev Ready 定义

满足以下条件可判定为 Dev Ready:

1. 按 `使用指南.md` 能在 15 分钟内完成本地启动。
2. 健康检查与就绪检查语义正确:
   - `GET /api/health` 返回 up。
   - `GET /api/health/readiness` 在依赖可用时返回 200。
3. 关键链路可用:
   - 库存创建/查询/调整。
   - Agent Chat + 任务状态查询。
   - Memory 写入/检索。
   - MCP tools/invoke。
4. 回归测试通过:
   - 单元测试通过。
   - 集成测试在依赖可用时通过。

## 4. 文档维护规则

1. 同一事实只在一个文档定义，其他文档仅引用。
2. 每次接口或配置变更后，必须同步修改对应文档。
3. 进度文档只记录未完成事项，完成项写入“现状总览”。
