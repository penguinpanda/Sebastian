# Sebastian 项目开发现状总览

更新时间: 2026-06-11

## 1. 文档目的

本文件是“当前已实现能力”的快照。未完成项与排期请看 `docs/开发进度与待办.md`。

## 2. 当前实现能力

### 2.1 工程基础

1. FastAPI + SQLAlchemy + Alembic 工程链路已落地。
2. Docker Compose 包含 postgres、redis、migrate、api、celery-worker、celery-beat。
3. Makefile 提供 `test`、`test-unit`、`integration` 统一入口。

### 2.2 业务能力

1. 库存 API 已完整支持创建、查询、调整、汇总、临期提醒。
2. Agent Chat 已支持任务入库、状态缓存、队列统计。
3. Agent Tool API 已提供 recipe、health、equipment、search 四类入口。
4. MCP API 已支持 tools 列表与 invoke 调用。
5. memory 搜索支持 lexical/vector/hybrid 与 RRF 融合。

### 2.3 可观测与异步

1. 请求 trace_id 透传/生成已启用。
2. readiness 端点已支持依赖异常返回 503。
3. Celery 过期扫描任务与任务历史查询 API 已落地。

### 2.4 数据与模型

1. 已落地表: inventories、inventory_transactions、agent_tasks、tool_call_logs、celery_task_execution_logs。
2. 内存仓储保留用于测试替身，默认实现为 PostgreSQL 仓储。

## 3. 已知限制

1. 当前缺少统一认证机制（更偏开发环境）。
2. 前后端 E2E 自动化覆盖不足。
3. 尚未建立完整性能基线与告警体系。
4. Elasticsearch/外部 LLM 不可用时，部分能力进入降级路径。

## 4. 近期验证状态

1. 单元/接口测试可稳定通过。
2. 集成测试可通过 Makefile 目标执行。
3. 最近关键链路测试通过: inventory graph 与 agent chat API。
